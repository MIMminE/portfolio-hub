# DevPilot 100% 완성 기획안

## 1. 제품 정의

DevPilot는 개발자가 매일 반복하는 Jira 확인, 저장소 정리, 브랜치 생성, Codex 작업 지시, 보고서 작성, 업무 기록을 한 흐름으로 묶는 macOS용 개인 개발 비서다. 목표는 단순 자동화 도구가 아니라 "오늘 어떤 일을 시작하고, 어디를 고치고, 무엇을 보고해야 하는지"를 한 화면에서 판단하게 만드는 것이다.

100% 완성 상태의 DevPilot는 다음 질문에 즉시 답해야 한다.

- 오늘 내가 해야 할 일은 무엇인가?
- 이 Jira 일감은 어떤 화면/기획/첨부와 관련되어 있는가?
- 어떤 repository와 branch에서 작업해야 하는가?
- Codex에게 어떤 맥락과 컨벤션으로 일을 시켜야 하는가?
- 오늘 한 작업과 보고서, 메모, 연장 근무 기록은 어디에 남는가?
- 문제가 생겼을 때 원문 오류를 쉽게 복사하고 복구 흐름으로 이어갈 수 있는가?

## 2. 완성도 기준

| 영역 | 100% 기준 |
| --- | --- |
| 대시보드 | 오늘 할 일, 진행 중 작업, 저장소 위험, Codex/인증 상태가 5초 안에 파악된다. |
| Jira 작업 | 일감 상세, 본문, 댓글, 첨부, 연결 repository, 브랜치, PR까지 한 흐름으로 이어진다. |
| 기획 자료 | Jira 첨부 이미지와 링크를 앱 안에서 확인하고, Codex 작업 지시 컨텍스트에 포함한다. |
| 저장소 | 브랜치 상태, 오늘 작업, 자동 rebase/pull 결과, PR/릴리즈 정보가 카드에서 명확히 보인다. |
| Codex | 앱 공통 프롬프트 규칙과 사용자의 요청을 합쳐 균일한 작업 지시를 만든다. |
| 보고서 | 초안 생성, Codex 다듬기, 제출, Slack 전송, 앱 기록 저장이 끊기지 않는다. |
| 기록 | 날짜별 보고서, 메모, Jira 흐름, 연장 근무가 타임라인/탭으로 재조회된다. |
| 설정 | 기본/연동/고급/점검이 분리되고, 자주 쓰지 않는 설정은 접힌 상태로 관리된다. |
| 오류 처리 | 모든 오류 모달에서 원문 복사, 복구 행동, 다음 단계 안내가 제공된다. |
| 배포 | 로컬 빌드, 앱 번들, 릴리즈 산출물, 공증 전/후 배포 전략이 문서화된다. |

## 3. 핵심 사용자 흐름

### 3.1 출근 후 첫 화면

1. 앱 실행 시 Jira, 저장소, 기록, Codex 상태를 병렬로 로드한다.
2. 대시보드는 오늘 할 일, 진행 중 브랜치, 정비 필요 repository, 최근 보고 상태만 보여준다.
3. 사용자는 할 일을 선택하거나, 저장소 위험 신호를 먼저 처리한다.

### 3.2 Jira 일감 시작

1. Jira 일감을 선택한다.
2. 상세 패널에서 기획 본문, 댓글, 첨부 이미지, 링크를 확인한다.
3. repository가 연결되어 있지 않으면 연결 창을 띄운다.
4. 연결된 repository 기준 브랜치에서 작업 브랜치를 생성한다.
5. Codex 작업 요청 창에 Jira 키, 본문, 첨부 요약, repository, 브랜치, 컨벤션을 자동 삽입한다.
6. 작업 시작 이벤트를 기록에 남긴다.

### 3.3 저장소 정리

1. 각 repository 카드는 기준 브랜치, 현재 브랜치, today activity, PR/릴리즈 정보를 보여준다.
2. 자동 처리 가능한 pull/rebase는 실행하고 결과를 "자동 처리됨"으로 남긴다.
3. 충돌/dirty/다중 기준 브랜치 같은 케이스는 버튼을 비활성화하고 이유를 알려준다.

### 3.4 보고서 제출

1. 오늘 커밋/머지/메모/Jira 처리 흐름을 수집한다.
2. 초안을 만들고 Codex 또는 ChatGPT 전달용 프롬프트로 다듬는다.
3. 제출 시 앱 기록에 저장하고, 업무 프로필에서는 Slack 전송도 선택한다.
4. 기록 화면에서 날짜별 보고서를 다시 열 수 있다.

## 4. 개발 로드맵

### Phase 1. 안정적인 데모 가능 상태

- Jira 상세 패널 추가: 본문, 링크, 댓글 일부, 첨부 목록 표시
- Jira 첨부 이미지/파일을 브라우저 또는 Finder로 여는 버튼 제공
- 저장소 미연결/다중 연결/dirty 상태의 작업 시작 UX 안정화
- 오류 모달에 원문 복사와 복구 액션을 표준화
- 기록 화면 탭별 비어 있음/로딩/오류 상태 정리

### Phase 2. 개발자 작업 보조 완성

- Jira 첨부/본문 기반 Codex 프롬프트 컨텍스트 생성
- repository별 `AGENTS.md` 또는 프로젝트 컨벤션 파일 탐색 및 프롬프트에 자동 포함
- 작업 브랜치 생성 후 Codex 작업 루트 열기, PR 초안 생성까지 연결
- 자동 rebase/pull 결과 이력 저장
- PR/릴리즈 조회 실패와 인증 상태를 카드에 명확히 표시

### Phase 3. 기록과 보고 고도화

- 하루 작업 타임라인: Jira 시작, 브랜치 생성, 커밋, PR, 보고서, 메모, 연장 근무를 시간순으로 표시
- 보고서 템플릿 프리셋: 짧은 보고, 팀 공유, 회고, 장애 정리
- 제출된 보고서 검색/필터/재사용
- 메모와 보고서 사이의 연결 강화

### Phase 4. 개인 프로젝트 모드 완성

- 업무/개인 프로필별 GitHub 계정, 토큰, repository root 분리
- 개인 프로젝트용 이슈 소스: GitHub Issues 또는 로컬 작업 보드
- 개인 프로필에서는 Slack/Jira 업무 기능을 숨기고 GitHub/Codex/메모 중심으로 축소

### Phase 5. 배포 품질

- 앱 아이콘, 빈 상태, 로딩 상태, 에러 상태 전체 polish
- 릴리즈 zip 검증 스크립트
- macOS 공증 전/후 배포 문서
- 스모크 테스트 체크리스트

## 5. Git 작업 컨벤션

커밋 메시지는 현재 repository 흐름에 맞춰 다음 형식을 사용한다.

```text
feat: 사용자 기능 추가
fix: 버그/예외 처리 수정
refactor: 구조 개선
docs: 문서/기획 업데이트
chore: 빌드/운영 보조 변경
```

작업 단위 원칙:

- 한 커밋은 하나의 사용자 가치 또는 하나의 안정화 목적만 가진다.
- macOS UI, CLI, 문서가 모두 필요한 기능은 CLI 기반 -> macOS 연결 -> UI polish 순서로 나눈다.
- 기능 개발 전 미커밋 안정화 변경은 먼저 `fix:`로 커밋한다.
- 각 개발 묶음마다 `just check`와 `just macos-app-build`를 통과시킨다.
- 앱 UI가 바뀌면 앱 번들을 갱신하고 재실행한다.

## 6. 즉시 착수할 작업

1. Jira 상세/기획 자료 패널의 데이터 모델과 CLI 조회 명령 추가
2. 작업 화면에서 선택된 Jira 일감 상세 패널 표시
3. 첨부 목록과 외부 열기 버튼 제공
4. 첨부/본문 기반 Codex 작업 요청 컨텍스트 연결
5. 위 작업을 `feat: Jira 상세 기획 자료 패널 추가` 커밋으로 정리
