# DevPilot 면접 대비 노트

이 문서는 DevPilot를 이력서나 포트폴리오에서 설명한 뒤 받을 수 있는 질문을 따로 학습하기 위한 노트입니다.

## 핵심 메시지

DevPilot는 Jira, Git, Codex, Slack, 보고서를 각각 자동화한 앱이 아니라, **일감 하나의 생애주기를 로컬 워크플로우로 관리하는 개발 업무 콘솔**이다.

설명할 때는 “AI로 프론트엔드 앱을 만들었다”보다 다음 방향이 더 좋다.

> 백엔드/시스템 개발 경험을 바탕으로 업무 흐름을 정의했고, AI를 활용해 macOS 앱, CLI, 외부 연동, 패키징까지 제품 형태로 확장했습니다.

## 예상 질문

| 질문 | 답변 준비 방향 |
| --- | --- |
| 왜 SwiftUI 앱과 Python CLI를 분리했나요? | UI와 자동화 로직의 책임을 나누기 위해서. SwiftUI는 macOS 네이티브 경험, Python CLI는 Jira/Git/Slack/보고서 자동화를 담당한다. CLI를 분리하면 앱 밖에서도 테스트하고 스크립트로 재사용하기 쉽다. |
| 왜 로컬 우선 구조로 설계했나요? | Jira 내용, 토큰, 로컬 경로, 개인 메모 같은 민감 정보가 많아서 외부 서버 없이 동작하는 구조가 적합했다. 포트폴리오 공개를 위해 마스킹 모드도 별도로 두었다. |
| Codex 결과를 그대로 신뢰하지 않기 위해 어떤 장치를 뒀나요? | AI 분석, repository 확정, workspace 준비, 테스트, 보고를 단계로 나눴고 각 단계에서 사람이 승인하거나 기록을 남기게 했다. 테스트 누락이나 보고 누락은 리스크로 표시한다. |
| 일감 단위 workspace를 둔 이유는 무엇인가요? | 여러 작업이 한 로컬 폴더에 섞이면 브랜치와 변경 파일 추적이 어려워진다. worktree와 컨텍스트 파일을 일감 기준으로 묶으면 작업 범위와 보고 근거를 나중에 다시 찾기 쉽다. |
| 외부 연동 실패는 어떻게 다루나요? | Jira, GitHub, Slack, Git, Codex는 각각 실패 가능성이 다르기 때문에 상태 조회와 실행 결과를 분리하고, 실패 이유를 사용자에게 보여주는 방향으로 설계했다. 핵심 흐름은 로컬 기록으로 유지한다. |
| 이 프로젝트에서 백엔드 개발자 역량은 어디에 드러나나요? | 도메인 흐름을 상태 모델로 나누고, 외부 API와 로컬 Git 상태를 통합하며, 배치성 보고/기록 흐름을 설계한 부분이다. UI 구현보다 workflow, 상태, 연동, 실패 처리 설계가 핵심이다. |
| AI 의존적인 작업이었다는 점을 어떻게 설명할 건가요? | AI는 구현 속도를 높이는 도구로 활용했고, 요구사항 분해, 상태 모델, 외부 연동 경계, 검증 기준은 개발 경험을 바탕으로 정의했다고 설명한다. 모르는 기술을 AI로 보완했지만 최종 구조와 검증 책임은 개발자가 가져갔다고 말한다. |
| 왜 단순 TODO 앱이 아니라 개발 업무 콘솔이라고 설명하나요? | 일감, repository, 브랜치, Codex 컨텍스트, 테스트 결과, 보고서가 연결되어 있기 때문이다. 단순 목록 관리가 아니라 실제 개발 작업의 전후 맥락을 보관한다. |

## 더 공부할 주제

- SwiftUI 상태 관리와 macOS 앱 lifecycle
- Python CLI 설계, subprocess 실행, 에러 모델링
- Git worktree, branch 전략, safe cleanup 방식
- Jira/GitHub/Slack API 인증과 실패 처리
- 로컬 앱 데이터 저장 방식, 토큰/민감 정보 보호
- AI 작업 컨텍스트 설계와 guardrail
- 패키징, notarization, GitHub Actions 기반 macOS 릴리즈

## 면접에서 피해야 할 표현

- “AI가 거의 만들어줬습니다.”
- “프론트엔드는 잘 모르지만 일단 만들었습니다.”
- “개인용이라 예외 처리는 많이 안 했습니다.”

대신 아래처럼 말한다.

- “AI를 활용해 구현 범위를 넓혔지만, workflow와 상태 설계는 직접 정의했습니다.”
- “기본 역량은 백엔드/시스템 개발이고, 그 경험을 바탕으로 프론트엔드와 패키징까지 확장했습니다.”
- “외부 연동 실패와 민감 정보 노출을 고려해 로컬 우선 구조와 마스킹 모드를 두었습니다.”
